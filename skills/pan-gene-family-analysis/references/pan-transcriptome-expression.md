## 1. 数据来源与设计 / Data & design

- **来源**:优先物种泛基因组/专用数据库收录的多材料 RNA-seq(承接 `gene-family-analysis`
  的取数原则);自测则覆盖关心的**组织 / 发育阶段 / 处理(尤其胁迫响应)**。
- **设计**:明确对比维度——组织特异、发育动态、处理响应(胁迫 vs 对照)、**材料间差异**
  (不同种质同一条件)。泛层的独特价值在"材料间 + PAV 身份"的交叉。

## 2. 跨材料的定量映射(核心难点) / Cross-accession quantification

不同材料基因组不同,**不能简单按各自基因 ID 比表达**,需统一参照系:

- **统一到 orthogroup / 参考成员**:把各材料的家族成员表达汇总/映射到共同的 orthogroup
  或参考基因上,保证"同一行"在不同材料间可比。
- **比对策略二选一**:① 各材料 reads 比对各自基因组再按 orthogroup 汇总;② 统一比对到
  参考(或非冗余泛基因集)定量——后者更可比,但参考缺失的 private 成员会漏掉,需补该材料
  自身定量。
- **private/dispensable 成员**:在没有该基因的材料里表达自然为 0(结构性缺失),作图与统计
  时要把"结构性缺失的 0"与"有基因但不表达的 0"区分标注,别混为一谈。

## 3. 标准化、变换与作图 / Normalization & plotting

- 标准化 **TPM / FPKM**(同时校正样本总 reads 与基因长度);批次/材料间差异大时注意批次效应。
- 热图前数值变换**首选 Z-score**(log2 亦可);**R / pheatmap** 出图 + `hclust` 聚类。
- 在热图行注释里标注每个成员的 **core/dispensable 身份**,直观看"可变成员是否表达模式也特殊"。

## 4. 与 PAV / 拷贝数 / 表型联动 / Integration

- **PAV × 表达**:private 成员是否仅在特定材料、特定胁迫下表达?提示其为材料特异的功能补充。
- **CNV × 表达**:拷贝数高的材料是否表达更高(剂量效应)?
- **表达 × 表型**:把胁迫诱导强、且与抗逆表型关联的成员列为重点候选(承接第 10 步关联)。
- **重点候选**:在特定处理/组织**特异表达**、或**仅部分材料存在且响应胁迫**的成员,接
  qRT-PCR 验证。
